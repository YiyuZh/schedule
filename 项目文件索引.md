## 分端维护入口

### 电脑端

- `电脑端任务记忆文档.md`
- `电脑端项目文件索引.md`
- `电脑端项目文件索引.json`
- `schedule_backend/任务记忆文档.md`
- `schedule_backend/项目文件索引.md`
- `schedule_backend/项目文件索引.json`

电脑端包含：

- `schedule_backend/`：FastAPI + SQLite 本地后端。
- `schedule_frontend/`：Vue 3 + Tauri 桌面前端。
- `start-dev.ps1` / `stop-dev.ps1`：Windows 一键启动和停止。

### 手机端

- `schedule_mobile/手机端操作手册.md`
- `schedule_mobile/任务记忆文档.md`
- `schedule_mobile/项目文件索引.md`
- `schedule_mobile/项目文件索引.json`

手机端包含：

- Vue 3 + TypeScript + Vite。
- Pinia + Vue Router。
- Capacitor。
- IndexedDB 本地优先。
- 云同步直连 `schedule_sync_server`。
- 当前已进入最终功能性基线：模板自动刷新每日任务、AI 保留单独开始时间、日程事件可编辑删除、push 成功后本地实体标记为已同步。
- 同步页和我的页已提供自动同步可观察状态：最近自动同步、下次重试、队列错误、待上传数量。
- 后续手机端只做紧急修复、iOS 打包发布和同步协议兼容；新业务优先更新电脑端和云服务器端。
- 最高约束：维护电脑端、云服务器端、部署脚本、Caddy 或管理后台时，默认不得修改 `schedule_mobile`。

## 验证命令

电脑端后端：

```powershell
cd D:\apps\schedule\schedule_backend
pytest -q
python -c "from app.main import app; print(app.title)"
```

电脑端前端：

```powershell
cd D:\apps\schedule\schedule_frontend
npm test -- --run
npm run build
npm run tauri:build
```

```powershell
cd D:\apps\schedule\schedule_mobile
npm test -- --run
npm run build
npm run cap:sync
```

云同步服务端：

```powershell

```
