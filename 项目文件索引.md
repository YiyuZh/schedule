## 电脑端

维护入口：

- `电脑端任务记忆文档.md`
- `电脑端项目文件索引.md`
- `电脑端项目文件索引.json`
- `schedule_backend/任务记忆文档.md`
- `schedule_backend/项目文件索引.md`
- `schedule_backend/项目文件索引.json`

包含：

- `schedule_backend/`：FastAPI + SQLite 本地后端。
- `schedule_frontend/`：Vue 3 + Tauri 桌面前端。
- `build-release.ps1`：NSIS 一体化安装包构建脚本。
- `release/Schedule_0.4.0_x64_setup.exe`：当前发布包。

## 手机端

维护入口：

- `schedule_mobile/手机端操作手册.md`
- `schedule_mobile/任务记忆文档.md`
- `schedule_mobile/项目文件索引.md`
- `schedule_mobile/项目文件索引.json`

约束：

- `schedule_mobile` 已冻结为最终功能性基线。
- 维护电脑端、云服务器、Caddy、部署脚本或管理员后台时，默认不得修改手机端。
- 后续新增功能优先在电脑端和云服务器端实现，并保持云端协议向旧手机端兼容。

## 云同步服务端

独立项目路径：

```text
D:/apps/schedule_sync_server
```

部署路径：

```text
/opt/apps/schedule-sync
```

正式域名：

```text
https://schedule-sync.zenithy.art
```

维护入口：

- `D:/apps/schedule_sync_server/README.md`
- `D:/apps/schedule_sync_server/任务记忆文档.md`
- `D:/apps/schedule_sync_server/项目文件索引.md`
- `D:/apps/schedule_sync_server/项目文件索引.json`

## 当前核心能力

- 电脑端本地业务完整可用。
- 电脑端可生成用户无依赖安装包。
- 云同步采用 sync envelope v1。
- AI Key 不进入云端。
- DeepSeek 是默认 AI 路线。
- 手机端为冻结基线，后续主要通过电脑端和云端兼容演进。

## 验证命令

电脑端后端：

```powershell
cd D:\apps\schedule\schedule_backend
pytest -q
python -c "from app.main import app; print(app.title)"
```

电脑端前端：

```powershell
cd D:\apps\schedule\schedule_frontend
npm test -- --run
npm run build:desktop
npm run tauri:build
```

发布包：

```powershell
cd D:\apps\schedule
.\build-release.ps1
```
