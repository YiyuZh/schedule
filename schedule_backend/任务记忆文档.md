## 当前状态

- 技术栈：FastAPI + SQLAlchemy 2.x + Pydantic v2 + SQLite
- 角色：Windows 电脑端本地业务后端
- 当前阶段：本地业务完整 MVP + 云同步客户端底座 + 三端同步 apply 第一版

## 关键决策

- 电脑端仍以本地 SQLite 为主，不依赖云端在线才能使用。
- 云同步只同步业务数据，不同步 AI Key、Token、本地运行配置。
- 本地写入成功后进入 `sync_queue`。
- 云端 accepted 后才删除队列；rejected/conflict 保留队列并写入错误。
- pull 远端数据时，本地干净则落库，本地 dirty 则进入 `sync_conflicts`。
- 跨设备关联字段使用 sync id 解析。

## 维护重点

- 新增业务表若要同步，必须同时补模型同步字段、`ensure_sync_schema()`、`sync_mapper`、service 入队和测试。
- 不要把业务逻辑写进路由层，继续保持 route -> service -> model。
- 保持 `/api/sync/*` 对前端稳定。
- SQLite 迁移当前采用启动时安全补列，不使用 Alembic。
- 测试环境常用 `autoflush=False`，涉及关系解析时要显式 `flush()`。
