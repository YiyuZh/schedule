## 当前状态

- 根目录：`D:\apps\schedule`
- 后端：`schedule_backend`
- 前端：`schedule_frontend`
- 桌面壳：`schedule_frontend/src-tauri`
- 技术栈：FastAPI + SQLite + Vue 3 + TypeScript + Tauri 2
- 当前阶段：电脑端可开发运行、可云同步、可生成 Windows NSIS 安装包。

## 关键决策

- 电脑端本地优先，核心业务运行在本地 FastAPI + SQLite。
- 云服务器只负责账号、设备、同步记录和变更日志，不承载电脑端业务逻辑。
- AI Key 只保存在本机，不进入同步队列、日志或云端 payload。
- 手机端 `D:\apps\schedule\schedule_mobile` 已冻结为最终功能性基线；维护电脑端时默认不得修改手机端。
- 普通用户发布形态固定为 NSIS 安装包 exe，不要求用户安装 Python、Node.js、Rust。
- 安装版本地后端默认监听 `127.0.0.1:18765`，开发版继续使用 `127.0.0.1:8000`。
- 安装版数据库固定在 `%APPDATA%\日程安排\data\app.db`，不把开发机 `data/app.db` 打包给用户。

## 一体化安装包

已新增发布链路：

- `build-release.ps1`
- `build-release.bat`
- `schedule_frontend/.env.desktop`
- `schedule_frontend/src-tauri/binaries/`
- `schedule_frontend/src-tauri/tauri.conf.json` 的 `bundle.externalBin`
- `schedule_frontend/src-tauri/src/main.rs` 自动启动 sidecar 后端

发布流程：

1. PyInstaller 将 `schedule_backend/run.py` 打成 `schedule-backend.exe`。
2. 脚本把后端 exe 复制为 Tauri sidecar：`schedule-backend-x86_64-pc-windows-msvc.exe`。
3. Vite 使用 `desktop` 模式构建，API 地址为 `http://127.0.0.1:18765`。
4. Tauri 构建 NSIS 安装器。
5. 最终安装器复制到 `release/Schedule_0.4.0_x64_setup.exe`。

Tauri 启动逻辑：

- 启动时检查 `127.0.0.1:18765/api/health`。
- 若已有可用日程安排后端则复用。
- 若端口被其他程序占用则弹出可读错误。
- 若无后端则启动 sidecar，并设置：
  - `SCHEDULE_HOST=127.0.0.1`
  - `SCHEDULE_PORT=18765`
  - `SCHEDULE_DATA_DIR=%APPDATA%\日程安排`
- 健康检查成功后显示主窗口。
- 退出时关闭 PyInstaller sidecar 进程树。

## 后续建议

1. 在一台干净 Windows 10/11 机器上安装 `release/Schedule_0.4.0_x64_setup.exe` 做真实用户验收。
2. 如需自动更新，后续可接 Tauri updater；第一版先采用手动下载新版安装包覆盖安装。
3. 增加安装包签名，减少 Windows SmartScreen 提示。
4. 后续电脑端新功能应优先通过云端协议兼容手机端，不再改手机端代码。
