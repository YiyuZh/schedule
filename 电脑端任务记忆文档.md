- 后端：`schedule_backend`
- 前端：`schedule_frontend`

- 技术栈：FastAPI + SQLite + Vue 3 + TypeScript + Tauri
- 当前阶段：本地业务 MVP 完成，Tauri 可构建，云同步客户端已接入三端同步闭环第一版；登录/注册后会自动 bootstrap 拉取云端已有数据

## 关键决策

- 电脑端本地优先，核心业务运行在本地 FastAPI + SQLite。
- 云服务器只同步数据，不承载电脑端业务逻辑。
- AI Key 只存本机，不进入云端 payload。
- 手机端 `D:\apps\schedule\schedule_mobile` 已进入最终功能基线冻结状态；维护电脑端时默认不得修改手机端代码、配置或文档。
- 电脑端新增业务字段或同步能力时，优先通过云端兼容旧手机端，不要求手机端同步改版。
- 前端通过本地后端 `http://127.0.0.1:8000` 调用业务 API。
- Tauri 只负责桌面壳、本地窗口和少量本地能力。
- 同步失败不阻塞本地 CRUD。
- 云端 `accepted_items` 返回后，电脑端会把本地实体标记为已同步并更新 `sync_version/last_synced_at`。
- 固定任务模板刷新每日任务使用确定性 sync id，避免手机端和电脑端重复生成同一天模板任务。
- 自动同步采用轻量准实时策略：本地写入同步队列后 3 秒防抖上传，前端前台每 45 秒拉取一次。
- 每日任务完成/取消完成状态使用 `daily_task_status` 后改覆盖，只同步状态、完成时间和实际时长，不覆盖标题/备注。

## 当前能力

- 今日任务、任务模板、日程、课程、学习记录。
- 长期任务、子任务和生成今日任务。
- 学习科目统计。
- AI 工作区与 DeepSeek 双模型。
- 课表 JSON 导入与 AI 转课表 Prompt。
- Tauri 开发和 NSIS 安装包构建。
- Windows 一键启动脚本。
- 云同步设置、注册登录、手动同步、冲突记录。
- 已登录状态下设置页隐藏密码输入，只有“重新登录/切换账号”时再显示密码表单。
- 顶部和后台自动同步成功后会触发当前页面刷新。

## 维护重点

- 后端业务逻辑继续放 service 层，不放路由层。
- 前端不要直接访问云端业务数据；电脑端业务走本地后端。
- 同步相关改动需要同时检查后端 `sync_service.py`、前端 `stores/sync.ts`、云端协议。
- Tauri 打包前必须先跑前端 build。
- 不要把 AI Key 写入同步队列、日志或导出文件。
