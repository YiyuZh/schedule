# 日程安排手机端 UI 设计模型 Prompt

## 使用方式

把本文完整复制给专门做 UI 设计的 AI 模型，让它生成手机端 UI 设计稿。目标是为 `schedule_mobile` 生成 iPhone 端界面，不是桌面端界面。

请设计模型只输出移动端 UI 方案和页面设计，不要写后端代码，不要设计无后端支撑的假功能。

## 项目背景

项目名称：日程安排

这是一个本地优先的个人日程、长期任务与学习管理工具。已有 Windows 桌面端，准备新增 iPhone 手机端。

手机端技术路线：

- Vue 3
- TypeScript
- Vite
- Capacitor
- Pinia
- Vue Router

云端只做账号和同步：

- 用户注册 / 登录
- 设备注册
- 数据 push / pull
- 同步冲突记录

AI 由用户端自己配置 API Key：

- 手机端设置 DeepSeek / OpenAI 兼容接口
- API Key 只保存在本机
- 云服务器不保存 AI Key
- 云服务器不代理 AI 请求

## 视觉方向

请采用：

- iOS 原生感
- 单手操作友好
- 底部 Tab 导航
- 卡片化信息组织
- 蓝灰 Productive Order 风格
- 高可读性数字和状态标签
- 轻阴影、强边界、低饱和色
- 适合长期使用，不要花哨

建议色彩：

```text
主色：深蓝 #005FB8
辅助蓝灰：#3D6270
背景：#F6F8FA
卡片：#FFFFFF
边框：#DCE1E8
成功：#2F8F5B
警告：#B15B00
危险：#C24141
文本：#1A1C1C
弱文本：#667085
```

字体：

- iOS 默认 San Francisco 风格
- 中文使用系统苹方风格

## 全局导航

底部 Tab 固定 5 项：

1. 今日
2. 长期
3. 学习
4. 日程
5. 我的

底部 Tab 要适合拇指点击，当前选中项使用蓝色高亮。

全局状态：

- 顶部可显示同步状态小标签
- 离线时显示“离线使用中”
- 有待同步数据时显示“待同步 N 条”
- 同步失败时显示红色轻提示

## 页面 1：登录页

目标：让用户连接自己的云同步账号。

元素：

- App Logo / 名称“日程安排”
- 云服务器地址输入框
- 邮箱输入框
- 密码输入框
- 登录按钮
- 注册入口
- “本机设备名”输入框或自动生成提示
- 错误提示区域

状态：

- 默认状态
- 输入校验失败
- 服务器不可达
- 登录中 loading
- 登录成功跳转 Today

注意：

- 不要出现游客云同步。
- 不要设计第三方登录。

## 页面 2：今日任务页

目标：手机端最高频页面。

元素：

- 顶部日期：今天 / 星期
- 同步状态小标签
- 今日概览卡：
  - 总任务
  - 已完成
  - 学习时长
  - 完成率
- 任务列表卡片
- 右下角悬浮 `+` 按钮

任务卡片字段：

- 完成勾选
- 标题
- 分类 / 学习科目
- 状态标签
- 优先级标签
- 计划 / 实际时长
- 来自长期任务标签

交互：

- 点击勾选完成 / 取消完成
- 点击卡片进入编辑
- 左滑显示删除 / 更多
- `+` 新建任务
- 下拉刷新触发同步

空状态：

- “今天还没有任务”
- 按钮：“新建任务”

错误状态：

- 同步失败但本地数据可用
- 保存失败

## 页面 3：新建 / 编辑任务页

元素：

- 标题输入
- 日期
- 分类
- 学习任务开关
- 如果学习任务开启，分类 label 改成“学习科目”
- 计划时长
- 优先级
- 备注
- 保存按钮

交互：

- 保存后返回 Today
- 必填标题
- 学习科目 placeholder：数学 / Python / 英语

## 页面 4：长期任务页

目标：管理跨周、跨月目标。

元素：

- 顶部标题“长期任务”
- 筛选：进行中 / 已完成 / 全部
- 长期任务卡片列表
- 右下角 `+` 新建长期任务

长期任务卡片字段：

- 标题
- 分类
- 起止日期
- 截止提醒：已逾期 / 今天截止 / N 天后截止
- 进度条
- 子任务完成数
- 优先级

点击长期任务进入详情。

## 页面 5：长期任务详情页

元素：

- 长期任务标题
- 描述
- 进度条
- 子任务列表
- 新增子任务按钮
- 编辑长期任务按钮

子任务卡片字段：

- 完成勾选
- 标题
- 分类 / 学习科目
- 截止日期
- 计划时长
- 状态
- “生成今日任务”按钮
- 已生成后显示“已加入今日”

交互：

- 完成 / 取消完成子任务
- 新增 / 编辑子任务
- 生成今日任务后提示成功

## 页面 6：学习记录页

目标：查看学习投入。

元素：

- 时间范围切换：今日 / 本周 / 本月
- 总学习时长
- 学习次数
- 科目时长排行
- 最近学习记录

科目排行：

- 数学：xx 小时
- Python：xx 小时
- 英语：xx 小时
- 用横向进度条展示

空状态：

- “还没有学习记录”
- 引导去完成学习任务

## 页面 7：日程页

目标：查看当天事件和课程。

元素：

- 日期切换
- 时间线
- 事件卡片
- 课程卡片
- 新增事件按钮

卡片字段：

- 标题
- 时间段
- 地点
- 分类
- 状态

第一版不要设计复杂拖拽。

## 页面 8：AI 设置页

目标：用户配置自己的 AI Provider。

元素：

- AI 启用开关
- Provider 选择：
  - DeepSeek 官方
  - OpenAI 兼容
  - 本地兼容服务
- Base URL
- Chat / Parse 模型
- Plan / Reasoner 模型
- API Key 输入
- 测试连接按钮

重要提示：

- “API Key 只保存在本机，不上传云服务器”
- “AI 请求由本机直接发送给模型服务”

状态：

- 未配置
- 测试中
- 连接成功
- 连接失败

## 页面 9：同步设置页

可放在“我的”里。

元素：

- 当前账号
- 当前设备名
- 云服务器地址
- 最近同步时间
- 待上传数量
- 冲突数量
- 手动同步按钮
- 退出登录按钮

状态：

- 已同步
- 同步中
- 离线
- 同步失败
- 有冲突

## 页面 10：同步冲突页

第一版可做简单列表。

元素：

- 冲突数量
- 冲突记录列表
- 本地版本摘要
- 云端版本摘要
- 操作：
  - 使用本地
  - 使用云端
  - 稍后处理

不要做复杂 diff。

## 页面 11：我的页面

元素：

- 用户邮箱
- 设备名
- 同步状态
- AI 设置入口
- 同步设置入口
- 关于应用
- 退出登录
