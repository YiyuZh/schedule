# 手机端构建方案

## 1. 目标

本方案用于从零开始构建 `schedule_mobile`，让“日程安排”具备 iPhone 端 App 能力。

手机端第一版目标：

- 使用 Vue 3 + TypeScript + Vite + Capacitor。
- 支持本地优先使用，弱网 / 离线时不丢数据。
- 通过云同步服务和 Windows 桌面端同步数据。
- 用户在手机端自行配置 AI API Key，手机端直接调用 DeepSeek / OpenAI 兼容接口。
- 云服务器不保存 AI Key，不代理 AI 请求。

第一版不追求完整复刻桌面端，而是做高频移动场景：

- 今日任务
- 长期任务
- 学习记录
- 日程列表
- 同步状态
- AI 设置
- 账号与设备管理

## 2. 推荐开发工具

### 2.1 Windows 当前阶段推荐

你现在在 Windows 上开发，推荐：

```text
IDE：VS Code
AI 编程：Codex CLI / Codex 插件
预览：Chrome DevTools Device Toolbar
联调：浏览器访问 Vite dev server
截图反馈：把手机尺寸预览截图发给 Codex 继续改
```

Codex 不能直接变成手机模拟器，但可以这样配合：

1. Codex 生成 / 修改手机端代码。
2. 你运行 `npm run dev -- --host 0.0.0.0`。
3. 在 Chrome 中打开页面。
4. 按 `F12`，切到 Device Toolbar，选择 iPhone 尺寸。
5. 截图发给 Codex，让 Codex 继续调整 UI。

如果手机和电脑在同一个局域网，也可以在 iPhone Safari 中访问：

```text
http://电脑局域网IP:5173
```

这样能提前感受真实手机屏幕尺寸。

### 2.2 真 iPhone App 阶段必须准备

iOS 真机运行、模拟器、签名、TestFlight 和 App Store 分发需要：

- Mac 或远程 macOS 构建环境。
- Xcode。
- Apple ID。
- TestFlight / App Store 需要 Apple Developer Program。

Capacitor 官方说明：Capacitor iOS App 由 Xcode 配置和管理，支持通过 `npx cap add ios` 添加 iOS 平台，通过 `npx cap open ios` 打开 Xcode 项目。参考：<https://capacitorjs.com/docs/ios>

Apple 设备和模拟器文档：<https://developer.apple.com/documentation/xcode/devices-and-simulator>

## 3. 为什么第一版选 Capacitor

推荐第一版选择 Capacitor，而不是 Tauri iOS 或 SwiftUI。

原因：

- 可以继续使用 Vue 3 / TypeScript。
- 可以先在 Windows 浏览器里开发大部分页面。
- 后续再用 Mac + Xcode 打包 iOS。
- 比 SwiftUI 重写更快。
- 比 Tauri iOS 更适合当前“先做移动端 MVP”的阶段。

Tauri 2 官方支持 iOS，但当前项目桌面端已经很重，第一版手机端更适合单独做轻量移动 UI。Tauri 可作为后续统一多端壳方案再评估。参考：<https://tauri.app/>

## 4. 从零创建项目

### 4.1 创建目录

建议放在项目根目录：

```powershell
cd D:\apps\schedule
npm create vite@latest schedule_mobile -- --template vue-ts
cd schedule_mobile
npm install
```

### 4.2 安装基础依赖

```powershell
npm install vue-router pinia
npm install @capacitor/core @capacitor/cli
npm install @capacitor/app @capacitor/network @capacitor/preferences
```

说明：

- `@capacitor/preferences` 只适合存非敏感配置。
- AI API Key 不建议用 Preferences 存，真 App 阶段应使用 iOS Keychain 或安全存储插件。
- 第一阶段如果只做 Web/PWA 预览，AI Key 可以临时存在浏览器本地，但必须标注为开发临时方案。

### 4.3 初始化 Capacitor

```powershell
npx cap init
```

建议填写：

```text
App name：日程安排
App ID：com.local.schedule
Web asset directory：dist
```

### 4.4 开发预览

```powershell
npm run dev -- --host 0.0.0.0
```

然后：

- 电脑浏览器访问 `http://127.0.0.1:5173`
- Chrome DevTools 选择 iPhone 尺寸
- 或 iPhone Safari 访问 `http://电脑局域网IP:5173`

### 4.5 iOS 平台添加

这一步需要 Mac：

```bash
npm run build
npm install @capacitor/ios
npx cap add ios
npx cap sync ios
npx cap open ios
```

在 Xcode 中选择 Simulator 或 iPhone 真机运行。

## 6. 页面规划

### 6.1 登录页

功能：

- 输入云服务器地址。
- 输入邮箱 / 密码。
- 登录。
- 可选注册入口。
- 显示当前设备名。

第一版必须有：

- 登录失败错误提示。
- 服务器不可达提示。
- 已登录状态自动进入 Today。

### 6.2 今日任务页

功能：

- 查看今日任务。
- 新增任务。
- 完成 / 取消完成任务。
- 补录实际时长。
- 标记学习任务。
- 显示同步状态。

移动端不要放太多按钮，优先：

- 底部悬浮 `+` 新建。
- 任务卡片左滑删除或更多操作。
- 点击卡片进入编辑。

### 6.3 长期任务页

功能：

- 查看长期目标。
- 查看子任务。
- 新增子任务。
- 完成 / 取消完成子任务。
- 子任务生成今日任务。

移动端交互：

- 上方为长期任务横向卡片或列表。
- 下方展示当前长期任务的子任务。
- 生成今日任务后显示“已加入今日”。

### 6.4 学习记录页

功能：

- 今日 / 本周 / 本月切换。
- 学习总时长。
- 按学习科目排行。
- 最近学习记录。

移动端第一版不做复杂图表，用条形排行和数字卡即可。

### 6.5 日程页

功能：

- 查看当天事件。
- 查看课程。
- 新增简单事件。
- 按时间线展示。

第一版不做复杂拖拽。

### 6.6 同步页

功能：

- 当前账号。
- 当前设备。
- 最近同步时间。
- 待上传数量。
- 待处理冲突数量。
- 手动同步按钮。
- 退出登录。

### 6.7 AI 设置页

功能：

- Provider 选择。
- Base URL。
- Chat Model。
- Plan Model。
- API Key。
- 测试连接。

重点：

- API Key 只保存在本机。
- 不上传云服务器。
- 页面需要明确提示这一点。

### 6.8 个人中心页

功能：

- 账号信息。
- 设备名。
- 同步设置。
- AI 设置入口。
- 关于应用。

## 7. 本地数据设计

第一阶段建议用 IndexedDB，后续真 App 再评估 SQLite。

本地表建议：

```text
local_entities
- entity_type
- entity_id
- payload
- sync_version
- sync_dirty
- sync_deleted
- updated_at

sync_queue
- id
- entity_type
- entity_id
- operation
- payload
- base_version
- retry_count
- last_error
- created_at

sync_state
- server_url
- access_token
- refresh_token
- device_id
- latest_change_id
- last_sync_at
```

注意：

- `api_key` 不放入这些同步表。
- AI Key 使用 `secureStorage` 单独封装。

## 8. 同步客户端设计

手机端同步流程：

```text
本地写入
  -> sync_queue
  -> push 到云服务器
  -> pull 云端变更
  -> merge 到本地
```

最小 API：

```text
POST /api/auth/login
POST /api/auth/refresh
POST /api/auth/logout
POST /api/devices/register
POST /api/sync/push
GET  /api/sync/pull
GET  /api/sync/bootstrap
```

自动同步策略：

- App 启动后同步一次。
- 用户下拉刷新时同步一次。
- 网络恢复时同步一次。
- 前台状态每 30 - 60 秒尝试一次。

## 9. 用户端 AI 设计

手机端 AI 不调用你的云服务器。

```text
手机端 App
  -> 读取本机 AI Key
  -> 调用 DeepSeek/OpenAI 兼容接口
  -> 得到 JSON 结果
  -> 写入本地数据
  -> 通过同步队列同步业务结果
```

同步到云端的是“任务、事件、规划结果被应用后的业务数据”，不是用户 AI Key。

## 10. Codex 开发工作流

建议每次让 Codex 做小批次：

1. 搭建 `schedule_mobile` 基础项目。
2. 做路由和底部 Tab。
3. 做登录页。
4. 做本地 IndexedDB 封装。
5. 做同步客户端。
6. 做 Today 页面。
7. 做长期任务页面。
8. 做学习记录页面。
9. 做 AI 设置页。
10. 做 Capacitor 集成。

每次迭代：

```text
Codex 改代码 -> npm run dev -> Chrome 手机尺寸预览 -> 截图 -> Codex 修 UI
```

如果需要自动化 UI 检查，可以后续加 Playwright，先用移动 viewport 截图。

## 11. 第一版验收

手机端 Web/PWA 阶段：

- 能登录。
- 能显示 Today。
- 能新增 / 完成任务。
- 能新增长期任务子任务。
- 能同步到云服务器。
- 电脑端能拉到手机端数据。
- 手机端能配置 AI Key 并测试连接。

iOS App 阶段：

- 能在 Xcode Simulator 运行。
- 能在 iPhone 真机运行。
- API Key 存本机安全存储。
- 退出登录后本地 token 清理。
- 离线新增任务，联网后同步。
