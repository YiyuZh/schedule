# 短期升级方案

## 1. 文档定位

这份文档用于约束 `D:\apps\schedule` 在当前阶段的短期升级顺序。  
优先目标不是继续无边界扩功能，而是把已经落地的 `AI 工作区 -> AI 日志 -> 应用结果` 这条真实使用链路做稳、做可回归、做可维护。

修改本项目时，先读：
1. `任务记忆文档.md`
2. `项目文件索引.md`
3. `项目文件索引.json`
4. 本文档

## 2. 当前基线

当前项目已经具备：

- FastAPI 后端
- Vue 3 + TypeScript 前端
- Pinia + Vue Router
- Tauri 桌面端
- DeepSeek 默认双模型接入
- `AI 工作区` 页面
- `长期任务` 页面
- 学习科目统计与科目时长排行
- Windows 启动脚本
- NSIS 安装包构建能力

当前验证状态：

- `schedule_backend`: `pytest -q` 通过
- `schedule_backend`: `python -m compileall app` 通过
- `schedule_frontend`: `npm test` 通过
- `schedule_frontend`: `npm run build` 通过
- `schedule_frontend`: `npm run tauri:build` 通过

## 3. 当前阶段判断

上一轮 AI 闭环已经完成。本轮新增的短期重点是把长期任务和学习科目统计这条真实使用链路收口：

1. 长期目标可以拆成子任务，并生成今日任务执行
2. 子任务完成后父任务进度必须稳定刷新
3. 学习记录页用“学习科目”表达数学、Python、英语等学习投入

## 4. 已完成能力

### 4.1 已完成的本轮升级

- 新增后端 AI apply 防呆校验：
  - 日志类型不匹配时拒绝应用
  - `parsed_success = false` 时拒绝应用
  - 已应用过的日志拒绝重复应用
  - `parse_apply` 对不支持的 `action_type` 给出明确错误
- 新增后端 AI 工作流回归测试：
  - `parse -> parse/apply`
  - `plan -> plan/apply`
  - 类型不匹配
  - 失败日志拒绝应用
  - 重复应用拒绝
  - 不支持动作类型拒绝
- 补强前端 `AI 工作区`：
  - `测试连接`、`解析应用`、`方案应用` 都改成显式包装，避免未处理 Promise 冒泡
  - 工作区首屏刷新失败时显示统一提示
  - 日志详情加载失败时显示独立提示
  - 日志详情加载失败时清空旧详情，避免误读陈旧数据
- 新增前端页面级回归测试：
  - 首屏加载
  - 首屏部分失败提示
  - 空解析拦截
  - 解析应用失败时保留预览
  - 日志详情失败态
- 为页面级测试补入 `@vue/test-utils`
- 更新前端测试配置，使 `Element Plus` 组件页面测试可运行

### 4.2 本轮验收结果

- 后端测试：通过
- 前端测试：通过
- 前端构建：通过

### 4.3 长期任务与学习科目统计收口

- 新增后端长期任务与子任务模型、schema、service、router
- 新增前端长期任务页面、store、API、类型与编辑弹窗
- 子任务可生成今日任务，Today 页显示“长期任务”来源标识
- 子任务完成后父长期任务进度会自动更新
- 学习记录页按学习科目统计，并增加科目时长排行
- 侧边栏“新建日程”菜单新增“新建长期任务”真实入口
- 修复 `LongTermTasksPage.vue` 生产构建错误
- 已验证 `pytest -q`、`npm test -- --run`、`npm run build`、`npm run tauri:build`

## 5. 当前痛点

虽然 AI 闭环的代码级稳健性已经明显提升，但仍有几类工作值得继续推进：

- 还缺一次“真实 DeepSeek 配置下”的人工冒烟闭环
- `AI 工作区` 目前仍以单页工作流为主，日志与结果的横向比较能力有限
- 页面级测试已起步，但覆盖面还没有扩到设置页、导入页和计划页的复杂交互
- 桌面端打包已重新验证通过，但真实 AI 配置下的桌面端人工体验仍未复查

## 7. 暂缓项

本轮不建议优先推进：

- WiX 中文本地化恢复
- 大范围页面重设计
- 新 habits 模块
- 新的同步 / 账号体系
- 大规模后端 API 扩张

这些都不是当前最能提升可用性和稳定性的事情。

## 8. 下一步建议

按优先级建议直接这样推进：

1. 用真实 DeepSeek 配置走一次完整 AI 闭环人工冒烟
2. 手动走一遍长期任务：创建目标 -> 创建子任务 -> 生成今日任务 -> 完成子任务
3. 在桌面端实际执行 AI 与长期任务工作流，确认体验与 Web 一致
4. 视真实使用结果，再决定是否增强日志详情、结果对比和重试体验
