# 手机端操作手册

手机端长期维护、开发调试、云同步联通、iOS 测试和上架建议，统一维护在：

```text
schedule_mobile/手机端操作手册.md
```

其中已经补充：

- 手机端最终功能基线冻结规则：后续电脑端、云服务器端、部署脚本或管理后台升级默认不得修改 `schedule_mobile`。
- 从 Git clone 到 Xcode 安装到 iPhone 的完整步骤。
- App 后续更新路线：开发期 Xcode 重装、测试期 TestFlight、正式期 App Store。
- iOS 不能让普通 App 自己下载并静默安装新版，只能在 App 内检查更新并跳转。

这样做的原因是手机端后续可能独立维护、独立打包、独立接入 iOS 流程，把操作手册放在 `schedule_mobile/` 内更方便 Codex 和开发工具直接读取。

推荐阅读顺序：

1. `schedule_mobile/任务记忆文档.md`
2. `schedule_mobile/项目文件索引.md`
3. `schedule_mobile/手机端操作手册.md`
4. `三端联通上线操作手册.md`
