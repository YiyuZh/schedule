## 手机端冻结约束

- `D:\apps\schedule\schedule_mobile` 已冻结。
- 后续维护电脑端、云服务器、部署脚本、Caddy 或管理员后台时，默认不得修改手机端目录。
- 新业务功能优先在电脑端和云服务器端实现。
- 云端新增字段必须兼容旧手机端：可忽略、可读取、不崩溃。
- 只有 iOS 打包发布、严重故障紧急修复、或云端无法兼容旧手机端时，才允许最小化修改手机端。

## 关键决策

- 云服务器只做账号、设备、同步记录和变更日志。
- 云服务器不保存、不代理、不同步任何 AI API Key。
- AI 由用户端自行配置 API Key，DeepSeek 为默认路线。
- 同步协议采用 `sync envelope v1`。
- 关联字段跨设备传输使用 `sync_id`，不使用本地数字 ID。
- 每日任务状态采用轻量准实时同步和后改覆盖策略。
- 手机端冻结后，电脑端和云端必须承担后续兼容责任。

## 电脑端发布方案

普通用户发布形态固定为一个安装包 exe：

```text
release/Schedule_0.4.0_x64_setup.exe
```

构建命令：

```powershell
cd D:\apps\schedule
.\build-release.ps1
```

发布包实现：

- 后端 FastAPI 用 PyInstaller 打包为 `schedule-backend.exe`。
- Tauri 2 将后端作为 sidecar 打入 NSIS 安装包。
- 应用启动时自动拉起后端。
- 安装版后端监听 `127.0.0.1:18765`。
- 开发版后端继续使用 `127.0.0.1:8000`。
- SQLite 数据库存放在 `%APPDATA%\日程安排\data\app.db`。
- 用户电脑不需要安装 Python、Node.js、Rust、pip、npm。
