## 关键决策

- 云服务器只做账号、设备、同步快照和变更日志。
- 云服务器不保存、不代理、不同步任何 AI API Key。
- AI 由用户端自行配置 API Key；DeepSeek 为默认路线。
- 同步协议采用 `sync envelope v1`：
  - `entity_type`
  - `sync_id`
  - `sync_version`
  - `sync_deleted`
  - `relation_sync_ids`
  - `data`
- 跨设备关联一律使用 sync id，而不是本地数字 ID。
- 第一版冲突策略：本地干净则自动应用远端；本地 dirty 则写入冲突记录，不静默覆盖。
- 云端 push 返回 `accepted_items` 后，电脑端和手机端都要把本地实体标记为已同步并更新 `sync_version/last_synced_at`。
- 自动同步采用轻量准实时策略：本地入队后 3 秒防抖上传，前台每 45 秒拉取一次，不使用 WebSocket/SSE/APNs。
- 每日任务状态同步采用后改覆盖：`daily_task_status` 只合并 `status/completed_at/actual_duration_minutes`，不覆盖标题、日期、分类、备注。
- 云同步服务端 PostgreSQL 用户统一使用 `autsky`；`DATABASE_URL` 必须写成 `postgresql+psycopg://autsky:<URL编码后的密码>@postgres:5432/schedule_sync`，不能再使用旧用户 `schedule_sync` 或旧邮箱用户。
- 旧阶段文档归档到 `初期构建md/`，根目录只保留当前维护入口。
- 手机端阶段性方案文件已删除，手机端维护入口只保留 README、任务记忆、文件索引和操作手册。
